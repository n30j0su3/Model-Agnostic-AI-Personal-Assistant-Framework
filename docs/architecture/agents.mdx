# Agents

Agents are specialized AI instances with distinct instructions and fresh context windows.

## Core Agents

The framework includes several pre-built agents to handle system tasks:

### `@session-manager`
> **Propósito**: Gestión de sesiones diarias, recordatorios y cierre de día.

Responsable del seguimiento de la productividad diaria, recordatorios y resúmenes de fin de día. Interactúa con el directorio `sessions/`.

---

### `@context-sync`
> **Propósito**: Sincronización de conocimiento MASTER.

Asegura que el conocimiento definido en `.context/MASTER.md` se propague correctamente a todos los archivos de contexto específicos de las herramientas.

---

### `@feature-architect`
> **Propósito**: Arquitecto de producto y guardián de la filosofía.

Arquitecta y valida las características del backlog, previniendo solapamientos y produciendo planes técnicos antes de la ejecución.

- **Referencia**: `docs/architecture/feature-architect.mdx`
- **Reporte**: `docs/architecture/feature-analysis-report.mdx`

---

### `@orchestrator`
> **Propósito**: Orquestador de tareas complejas.

Coordina solicitudes multi-paso, delega en agentes/skills y consolida resultados.

- **Ubicación**: `agents/core/orchestrator/AGENT.md`
- **Bitácora**: `logs/orchestrator/`

---

### `@github-deployer`
> **Propósito**: Automatización de Git/GitHub.

Automatiza el flujo de trabajo de git, creando commits descriptivos y subiendo cambios a tu repositorio remoto.

## Creating Custom Agents

You can create your own specialized agents in the `agents/custom/` directory. Each agent should have an `AGENT.md` file following the system's template.

```markdown
---
name: your-agent-name
description: What the agent does
scope: workspace-name or global
tools: [list of tools]
---
# Instructions
...
```
