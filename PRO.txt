Vamos a generar un proyecto llamado "Model-agnostic AI Personal Assistant" donde vamos a crear un Framework de trabajo "estándar" para interactuar con la IA en modo "Personal Assistant". Crea el plan de trabajo en "plan-trabajo-v0.md" considerando lo siguiente:

El objetivo del framework es poder gestionar, optimizar y mejorar la productividad (personal y de trabajo) utilizando las mejores practicas opensource.
El framework debe ser lo suficientemente flexible para permitir que el usuario lo utilice para tareas multidisciplinarias (Ej: seguimiento de proyectos personales, trazabilidad de conversaciones y objetivos profesionales, investigación científica, creación de contenido, etc.) manteniendo separado el contexto de cada actividad.

El Framework debe cumplir con las siguientes características:
- Ser compatible con las principales "terminales-AI" (OpenCode - Gemini CLI - Claude Code - Codex)

- TODO el contexto del proyecto debe ser contexto almacenado localmente en archivos .md
(https://github.com/theNetworkChuck/ai-in-the-terminal/blob/main/docs/07-context-files.md)

- Debe permitir trabajar simultáneamente con las principales terminales AI en el mismo proyecto
(https://github.com/theNetworkChuck/ai-in-the-terminal/blob/main/docs/08-multi-tool-workflow.md)

- Debe ser compatible con "Skills" y múltiples AGENTS.md por feature/tarea como en este vídeo (https://www.youtube.com/watch?v=Nvn6s3r9ZAw&) :
1️⃣ **AGENTS.md**: Cómo escribirlo para que la IA no alucine
2️⃣ **División por features**: Múltiples agentes especializados
3️⃣ **Skills**: Sistema de habilidades que los agentes pueden invocar
4️⃣ **Skill Sync**: Script que sincroniza todo automáticamente

- La jerarquía del proyecto debe tener sentido lógico, semántico y ser intuitivo según el uso del usuario.

- Debe permitir /actualizar/gestionar la trazabilidad y cambios con GitHub (cada deploy)

- Debe tener un sistema de "sesión del día" que le permita al usuario: tener la trazabilidad, recordatorios de pendientes, cierre de trabajo del día y similares DOCUMENTADO en el contexto del proyecto/agentes. El usuario interactuará en lenguaje natural con este sistema. EJ: ("Qué proyectos tengo pendientes?", "Cuales son las tareas del día/semana/mes?", "Vamos a cerrar el día por hoy", etc.)

- Si una instrucción no es lo suficientemente clara, no encuentras información o falta contexto PIDE más información al usuario.

- Utilizar estándares Opensource:
  - For building professional UI/UX across multiple platforms and frameworks: (https://github.com/nextlevelbuilder/ui-ux-pro-max-skill)
  - For documentation: (https://www.mintlify.com/docs)
  - For SKills: (https://agentskills.io/specification)
  - For traducciones (https://www.i18next.com/overview/getting-started)

Ten en cuenta todos los requisitos necesarios para el funcionamiento del framework (documenta todo para guiar efectivamente al usuario). Como por ejemplo:
- Cuenta de GitHub
- Opencode
- Cuenta de Google (para usar Gemini CLI)
